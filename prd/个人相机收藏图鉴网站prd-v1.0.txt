## 一、文档概述
### 1.1 文档目的
本文档完整定义「个人相机收藏图鉴网站」的一期全部需求、功能规则、技术规范、验收标准，**可直接输入AI大模型，生成符合需求的完整网站代码包、部署操作指南、后台维护手册**，无模糊表述、无需求遗漏。
### 1.2 项目背景
需求方为个人相机收藏爱好者，需要搭建一个纯静态、零代码维护、国内访问稳定的个人相机图鉴网站，核心用于系统化收集、整理、查阅相机参数信息，仅个人与身边朋友使用，无需面向大众用户。
### 1.3 读者范围
AI代码生成工具、需求方本人、后续迭代开发参考
### 1.4 项目核心目标
1.  搭好完整的前台展示+后台管理框架，核心的相机数据增删改查功能全量跑通
2.  实现后台全可视化配置，无需修改代码/文件，即可完成网站分类、筛选维度、基础信息的调整
3.  部署在Gitee Pages，国内访问稳定，零服务器成本、零代码维护门槛
4.  优先满足个人查阅需求，界面极简专业，核心突出相机参数信息，无多余功能

## 二、核心定位与项目边界
### 2.1 网站核心定位
个人收藏专属相机图鉴，核心用途为系统化收集、整理相机参数信息，兼顾个人收藏记录属性，仅做非盈利性个人内容展示。
### 2.2 核心目标受众
需求方本人、身边的相机爱好者朋友，不面向大众公域用户。
### 2.3 一期项目验收边界
以下所有功能全部完成且符合规则，即视为一期项目验收通过：
1.  前台完整落地：首页、2个初始一级分类独立列表页、全量筛选/搜索/排序/视图切换功能，电脑端优先优化+手机端响应式适配
2.  后台完整落地：全可视化零代码操作，覆盖分类配置、字段配置、数据增删改查、批量导入/备份/恢复全功能
3.  数据同步正常：后台修改可正常通过Gitee API同步到仓库，前台可正常更新展示
4.  部署正常：代码可正常部署到Gitee Pages，国内主流浏览器访问稳定流畅
### 2.4 一期绝对不开发的功能（严格锁死边界）
1.  不做用户注册、登录、多用户管理功能
2.  不做任何电商售卖、付费内容、经营性相关功能
3.  不做评论、留言、分享等互动类功能
4.  不做小程序、APP等网页之外的其他端开发
5.  不做镜头、使用技巧、测评内容等非相机本身的扩展模块
6.  不做复杂的数据统计、分析类功能
7.  不做相机详情页，所有参数信息直接在列表页完整展示
8.  不做IE等老旧浏览器兼容，仅适配Chrome、Edge、360等主流现代浏览器

## 三、技术方案规范（AI生成代码必须严格遵守）
### 3.1 最终技术栈
| 模块 | 技术规范 | 硬性要求 |
|------|----------|----------|
| 前端开发 | 纯HTML + CSS + 原生JavaScript | 禁止使用任何前端框架（Vue/React等），无第三方框架依赖，保证代码极简、加载速度快 |
| 数据存储 | 单JSON文件（data.json）全量存储所有网站配置、相机数据 | 数据与代码完全分离，所有前台展示内容、后台配置均从该文件读取 |
| 部署平台 | Gitee Pages 纯静态托管 | 无需后端服务器、无需数据库服务，纯静态文件部署 |
| 数据同步 | Gitee 官方开放API | 前端JS通过API直接读取/修改仓库内的data.json文件，实现后台数据修改实时同步到仓库 |
| 后台权限 | 固定密码硬编码登录 | 无需复杂权限体系，极简登录逻辑 |

### 3.2 核心技术规则
1.  代码必须全量原生开发，无冗余依赖，保证电脑端首屏加载时间≤2秒
2.  必须做响应式布局，优先优化电脑端宽屏展示，同时完美适配手机端
3.  必须实现图片懒加载，优化页面加载速度
4.  Gitee API调用必须封装完整的异常处理，操作成功/失败有明确的页面提示
5.  Token、密码等敏感配置必须单独放在config.js配置文件中，标注清晰的替换说明，用户仅需修改该文件即可完成配置
6.  所有代码必须有清晰的注释，方便后续迭代修改

## 四、前台用户端功能需求
### 4.1 整体页面架构
| 页面名称 | 页面路径 | 核心用途 |
|----------|----------|----------|
| 首页 | index.html | 网站入口，核心分类导航、总览信息展示 |
| 胶片PS相机列表页 | film-ps.html | 胶片PS相机全量机型展示、筛选、搜索、排序 |
| CCD相机列表页 | ccd.html | CCD相机全量机型展示、筛选、搜索、排序 |
| 后台登录页 | admin-login.html | 后台管理入口，密码登录 |
| 后台管理页 | admin.html | 全功能可视化后台管理，仅登录后可访问 |

### 4.2 首页功能需求
#### 4.2.1 核心模块（按从上到下排序）
1.  顶部导航栏
    - 固定展示网站标题（后台可配置）、首页、一级大分类入口、后台登录入口
    - 响应式适配：手机端折叠为汉堡菜单
2.  网站核心总览区
    - 展示网站一句话介绍（后台可配置）、个人署名（后台可配置）
    - 展示核心数据总览：已收录机型总数、覆盖品牌总数
3.  最近更新机型区
    - 展示最近新增/修改的6台机型，按更新时间倒序排列
    - 每个机型卡片展示：机型缩略图、品牌+机型全称、发布年份、更新时间
    - 点击卡片直接跳转到对应大类型的列表页
4.  一级大分类快速入口区
    - 卡片式展示所有启用的一级大分类（初始为「胶片PS相机」「CCD相机」）
    - 每个分类卡片展示：分类名称、该分类下的机型总数
    - 点击卡片直接跳转到对应分类的独立列表页
5.  全局搜索框
    - 固定在首页显眼位置，支持输入品牌/机型名称，搜索全量机型
    - 搜索结果直接展示匹配的机型列表，点击跳转到对应分类的列表页
6.  页脚
    - 极简设计，无版权声明、无多余内容，仅展示网站更新时间

#### 4.2.2 交互规则
- 所有可点击元素有轻微hover高亮动效，不影响加载速度
- 页面滚动流畅，无卡顿
- 所有分类入口、跳转链接必须正常跳转，无死链

### 4.3 分类独立列表页功能需求（胶片PS相机/CCD相机共用一套逻辑，后台可独立配置）
#### 4.3.1 核心模块（按从上到下排序）
1.  顶部导航栏：与首页保持一致，高亮当前所在分类
2.  页面顶部控制区
    - 视图切换按钮：前台一键切换「卡片视图」/「表格视图」，后台可配置默认展示视图
    - 排序下拉框：可选排序规则「按发布年份正序/倒序、按更新时间倒序、按品牌首字母正序」，默认按发布年份倒序
3.  筛选区
    - 二级品类筛选：后台为该分类配置的二级品类选项，支持多选筛选
    - 品牌筛选：该分类下所有已收录的品牌，支持多选筛选
    - 自定义参数筛选：后台为该分类配置的可筛选参数字段（比如光圈、焦段、发布年份等），支持区间/多选筛选
    - 重置按钮：一键清空所有筛选条件，恢复全量列表
4.  机型列表展示区
    - 卡片视图：
      - 网格布局，电脑端默认4列，手机端默认1列，响应式自适应
      - 每个卡片固定展示内容：机型缩略图、品牌+机型全称、发布年份、二级品类、核心参数标签、个人备注缩略内容
      - 所有参数信息完整展示在卡片内，无需点击跳转详情页
    - 表格视图：
      - 电脑端多列完整展示，手机端横向滚动适配
      - 固定列：品牌、机型全称、发布年份、二级品类，其余列按后台配置的参数字段自动生成
      - 每行对应一台机型，所有参数完整展示，无需点击跳转
5.  空状态提示：筛选/搜索无匹配结果时，展示清晰的空状态提示

#### 4.3.2 交互规则
- 筛选条件选择后，列表实时刷新，无需点击确认按钮
- 视图切换、排序切换后，列表实时刷新，保留当前筛选条件
- 所有筛选、排序、视图切换操作，页面无刷新，前端实时渲染
- 图片懒加载，滚动到对应位置再加载图片，优化加载速度

## 五、后台管理端功能需求
### 5.1 整体权限规则
1.  后台登录采用固定密码验证，密码硬编码在config.js配置文件中，用户可自行修改
2.  未登录状态下，直接访问admin.html自动跳转到登录页
3.  登录成功后，记录本地缓存，关闭浏览器后缓存失效，下次打开需重新登录
4.  所有关键操作无需二次确认弹窗，简化操作流程

### 5.2 后台核心功能模块
#### 5.2.1 网站基础配置模块
1.  可配置项：
    - 网站标题、首页一句话介绍、个人署名
    - 前台列表默认展示视图（卡片/表格）
    - 后台登录密码
    - Gitee配置项：用户名、仓库名、私人令牌Token
2.  操作规则：
    - 修改后点击「保存配置」，自动同步到仓库的data.json文件
    - 保存成功后，前台1-2分钟内自动同步更新

#### 5.2.2 一级大类型管理模块（对应前台的一级分类）
1.  核心功能：
    - 新增一级大类型：填写类型名称、排序序号，启用/禁用状态
    - 编辑现有类型：修改类型名称、排序序号、启用/禁用状态
    - 删除类型：仅可删除无对应机型数据的类型
    - 调整排序：拖拽调整类型的前台展示顺序
2.  操作规则：
    - 禁用的类型，前台不展示入口，不影响后台已有的机型数据
    - 每个类型对应独立的列表页、独立的参数字段配置、独立的二级品类配置
    - 操作后点击「保存」，自动同步到仓库的data.json文件

#### 5.2.3 分类字段与筛选配置模块
1.  核心功能：
    - 先选择要配置的一级大类型，独立配置互不干扰
    - 通用字段配置：配置该类型下的通用必填字段，默认预设「品牌、机型全称、发布年份、二级品类、机型图片URL、个人备注」，可新增/删减/修改必填状态
    - 自定义参数字段配置：为该类型新增专属参数字段，可配置字段名称、字段类型（文本/数字/下拉选项）、是否必填、是否作为前台筛选项
    - 二级品类配置：为该类型配置独立的二级品类下拉选项，可新增/修改/删除选项
    - 前台筛选配置：配置该类型列表页的筛选项展示顺序、启用/禁用状态
2.  操作规则：
    - 字段类型为「下拉选项」的，必须配置固定选项值，前台/后台仅可从预设选项中选择
    - 字段类型为「数字」的，必须固定格式规范，比如发布年份仅可输入4位数字
    - 配置修改后，前台的列表展示、筛选功能自动适配，无需修改代码
    - 操作后点击「保存」，自动同步到仓库的data.json文件

#### 5.2.4 相机数据管理模块
1.  核心功能：
    - 先选择对应的一级大类型，展示该类型下的所有机型数据列表
    - 新增机型：弹出表单，按该类型配置的字段展示填写项，必填项标注清晰，填写完成后点击保存，自动新增到data.json中
    - 编辑机型：对现有机型进行内容修改，表单与新增保持一致，修改后点击保存自动同步
    - 删除机型：单条删除对应机型数据，无需二次确认
    - 搜索查询：在后台列表中按品牌/机型名称搜索对应机型
2.  操作规则：
    - 表单填写时，下拉选项字段仅可选择预设值，数字字段仅可输入符合格式的内容
    - 机型图片仅需填写图片URL地址，无需后台上传功能
    - 操作后点击「保存」，自动同步到仓库的data.json文件

#### 5.2.5 批量数据导入模块
1.  核心功能：
    - 先选择对应的一级大类型，再选择该类型下的目标品牌，按「大类型→品牌」分层导入
    - 提供标准JSON模板下载入口，模板严格匹配该类型配置的字段格式
    - 支持上传符合标准格式的JSON文件，一键批量导入对应品牌的机型数据
    - 导入前校验数据格式，格式错误给出明确的错误提示，不执行导入；格式正确则展示导入预览，确认后执行导入
2.  操作规则：
    - 导入的机型数据自动归属到选择的大类型和品牌下
    - 重复机型（品牌+机型全称完全一致）自动覆盖原有数据，不重复新增
    - 导入完成后给出明确的成功/失败数量提示

#### 5.2.6 数据备份与恢复模块
1.  核心功能：
    - 一键备份：点击「一键备份全量数据」，自动下载当前完整的data.json文件到用户本地电脑
    - 一键恢复：支持本地上传备份的data.json文件，预览恢复内容后，点击确认一键覆盖仓库中的data.json文件，完成数据恢复
2.  操作规则：
    - 恢复前必须展示预览内容，确认后再执行覆盖操作
    - 备份/恢复操作完成后给出明确的成功/失败提示

## 七、UI/视觉与交互规范
1.  整体风格：极简专业数码风，界面干净无冗余装饰，核心突出相机参数与图片信息，无多余视觉干扰
2.  配色规范：以深灰/炭黑为主色，纯白/浅灰为背景色，仅用低饱和品牌色做功能高亮，保证长时间浏览不刺眼，参数信息辨识度拉满
3.  交互规范：仅添加极轻微的卡片hover高亮、筛选切换顺滑过渡动效，不影响页面加载速度，核心优先保证操作流畅、无延迟
