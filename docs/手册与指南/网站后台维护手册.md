# 网站后台维护手册

## 1. 登录与退出

- 入口：`admin-login.html`。
- 密码字段：`data.json` → `siteConfig.password`（可在后台「网站基础配置」修改）。
- 登录态保存在 **sessionStorage**：**关闭浏览器后需重新登录**。

## 2. 网站基础配置

可配置：站点标题、一句话介绍、署名、列表默认视图（卡片/表格）、后台密码、GitHub `owner/repo/branch`（写入 `data.json`）。

**GitHub PAT**：不要写进会提交到 GitHub 的 `config.js`（公开仓库会触发 Secret scanning）。请使用 **`setup-token.html`** 在本机浏览器保存 Token；换浏览器或清除站点数据后需重新保存。

## 3. 一级大类型（分类）

- 新增/编辑/启用/禁用/排序（拖拽表格行左侧 **≡** 手柄）。
- **删除**仅当该分类下**没有**任何机型数据时允许。
- 禁用后：前台导航与入口隐藏，但后台数据仍保留。

## 4. 字段与筛选

- 先选择分类，再维护：
  - **二级品类**：每行一个选项。
  - **字段表**：`fieldId` 创建后尽量避免改名，以免影响历史 `specs` 数据。
  - 类型：`text` / `number` / `select` / `textarea`；`select` 的选项用英文逗号分隔。
  - **必填**、**是否作为前台筛选项**、行内 **↑/↓** 调整顺序（影响表单与筛选展示顺序）。
- **发布年份**：前台/导入校验为 **4 位数字**。

## 5. 相机数据

- 先选择分类；支持品牌/机型关键字过滤列表。
- **新增/编辑**：图片仅支持 **URL**（不上传文件）。
- **删除**：无二次确认，请谨慎操作。

## 6. 批量导入

1. 选择「一级大类型」与「目标品牌」。
2. 下载 `templates/import-template.json` 作为结构参考。
3. 上传符合格式的 JSON 数组；导入前会展示预览片段。
4. **重复规则**：同一分类下 **品牌 + 机型全称** 相同则 **覆盖**。

## 7. 备份与恢复

- **一键备份**：下载当前完整 `data.json`。
- **一键恢复**：选择本地 `data.json`，预览后覆盖仓库版本。
- **PAT 定期更新**：Fine-grained Token 到期会导致保存失败，请提前更新 `config.js` 中的 Token。

## 8. 常见问题排查

| 现象 | 排查 |
|------|------|
| 保存失败 | Token、分支名、仓库名是否匹配；`useLocalDataJson` 是否为 `false` |
| 前台不更新 | Pages 缓存延迟；是否保存到正确分支 |
| 无法登录 | 密码是否已改；是否加载了最新 `data.json` |

## 9. 数据兜底（误操作）

在 GitHub 仓库 **Commits** 中找到 `data.json` 的历史提交，使用 GitHub 提供的对比/恢复入口回滚（网页端即可查看历史）。
